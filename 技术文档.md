# 表格转换工具 — 技术文档

## 1. 项目概述

表格转换工具是一个纯前端、单文件 Web 应用，在浏览器中本地运行，无需安装、无需后端服务。用户通过拖拽或点击选择文件，即可实现表格文件（Excel/CSV）与纯文本文件（CSV/Markdown/固定宽度文本）之间的双向转换。

| 属性 | 值 |
|------|-----|
| 运行环境 | 现代浏览器（Chrome / Edge / Firefox） |
| 文件结构 | 单 HTML 文件，内嵌 CSS 和 JavaScript |
| 外部依赖 | SheetJS（xlsx）社区版 v0.20.3，通过 CDN 加载 |
| 数据处理位置 | 浏览器本地（不上传任何数据） |
| 支持的表格格式 | .xlsx / .xls / .csv |
| 支持的文本格式 | CSV / TSV / Markdown 表格 / 固定宽度文本 |

数据流向：用户文件 → FileReader API → JS 模块处理 → Blob 下载

### 2.2 模块划分

所有模块位于 `TableConverter` 命名空间下，采用 IIFE 模式隔离作用域。

| 模块 | 职责 | 关键方法 |
|------|------|----------|
| `Utils` | DOM 操作、事件绑定、Toast 提示、文件下载 | `qs()`, `on()`, `showToast()`, `triggerDownload()`, `getBaseName()` |
| `FileIO` | 文件读取与编码检测 | `readAsArrayBuffer()`, `readAsText()` |
| `FormatDetector` | 文本格式自动识别 | `detect()` |
| `TypeInferrer` | 列数据类型推断 | `inferColumnTypes()` |
| `Parsers` | 各格式文本解析为二维数组 | `autoParse()`, `parseDelimited()`, `parseMarkdown()`, `parseFixedWidth()` |
| `Converters` | 二维数组转换为各格式文本 | `toCSV()`, `toMarkdown()`, `toFixedWidth()` |
| `ExcelIO` | Excel 文件读写（封装 SheetJS） | `readExcel()`, `writeExcel()` |
| `App` | 应用入口，UI 初始化与事件协调 | `init()` |

**模块依赖关系：**

```
App ──┬── FileIO
      ├── FormatDetector
      ├── Parsers ──── FormatDetector
      ├── Converters
      ├── ExcelIO ──── XLSX (SheetJS CDN)
      ├── TypeInferrer
      └── Utils
```

## 3. 核心算法

### 3.1 格式自动检测（FormatDetector.detect）

**输入**：原始文本字符串

**输出**：格式描述对象 `{ type, delimiter, hasHeader }`

**流程：**

```
1. 取前 50 行（取前 20 行非空行作为分隔符分析样本）
2. Markdown 检测
   ├─ 条件 A：行中含 | 字符
   └─ 条件 B：存在分隔行（仅由 |、-、:、空格组成且至少包含一个 -）
   → 两条件同时满足 → 判定为 Markdown
3. CSV/TSV 检测
   ├─ 对逗号、Tab、分号分别统计每行出现次数（排除引号内）
   ├─ 过滤有效候选（≥50% 行中出现）
   ├─ 计算每个候选的"得分"（方差 + 平均频率惩罚）
   └─ 选取得分最低的分隔符
4. 固定宽度检测
   ├─ 记录每行中连续 2+ 空格的位置区间
   ├─ 对每个列位置聚类（±3 字符容差）
   └─ 出现在 ≥60% 行中的位置 → 判定为列边界
5. 兜底 → CSV
```

**分隔符稳定性计算：**

- 方差越小 → 每行字段数越一致 → 更可能为该分隔符
- 附加惩罚 `1/(avg+1) * 0.1` → 在方差接近时偏好出现频率更高的分隔符

### 3.2 数据类型推断（TypeInferrer.inferColumnTypes）

**输入**：数据行数组（不含表头）、表头数组

**输出**：列类型数组 `[{ name, type: 'number'|'date'|'text', sample }]`

**阈值**：≥80% 的非空样本值匹配某模式时，判定为该类型。

**数值检测正则（按顺序）：**

| 模式 | 正则 | 示例 |
|------|------|------|
| 整数/小数 | `/^-?\d+\.?\d*$/` | `123`, `-45`, `3.14` |
| 科学计数 | `/^-?\d+\.?\d*[eE][+-]?\d+$/` | `1.5e10` |
| 千分位 | `/^-?\d{1,3}(,\d{3})*(\.\d+)?$/` | `1,234,567.89` |
| 百分比 | `/^\d+%$/` | `85%` |
| 美元金额 | `/^\$\s*\d{1,3}(,\d{3})*(\.\d+)?$/` | `$1,234` |
| 人民币金额 | `/^￥\s*-?\d{1,3}(,\d{3})*(\.\d+)?$/` | `￥1,234` |
| 带"元"后缀 | `/^-?\d{1,3}(,\d{3})*(\.\d+)?\s*元$/` | `1,234元` |

**日期检测正则：**

| 模式 | 示例 |
|------|------|
| `yyyy-MM-dd` 或 `yyyy/MM/dd` | `2024-01-15` |
| `MM/dd/yyyy` | `01/15/2024` |
| 中文日期 | `2024年1月15日` |
| 带时间 | `2024-01-15 14:30` |

日期检测除正则以外部使用 `new Date().getTime()` 验证有效性，并排除纯数字（如 `2024`）和超出 1900–2100 年的值。

### 3.3 CSV 解析（Parsers._splitCSVLine）

遵循 RFC 4180 标准：

- 字段按分隔符切分
- 双引号包裹的字段内部可包含分隔符、换行符
- 引号内的连续两个引号 `""` 转义为一个引号
- 解析后 trim 每个字段

### 3.4 固定宽度输出对齐（Converters._displayWidth）

字符显示宽度计算规则：

| 字符范围 | Unicode 范围 | 宽度 |
|----------|-------------|------|
| ASCII 字符 | 0x00–0x7F | 1 |
| CJK 统一表意文字 | 0x4E00–0x9FFF | 2 |
| CJK 扩展 A | 0x3400–0x4DBF | 2 |
| CJK 扩展 B+ | 0x20000–0x2FFFF | 2 |
| 全角符号/字母 | 0xFF00–0xFFEF | 2 |
| CJK 符号标点 | 0x3000–0x303F | 2 |
| CJK 部首补充 | 0x2E80–0x2FDF | 2 |
| CJK 兼容汉字 | 0xF900–0xFAFF | 2 |
| CJK 兼容形式 | 0xFE30–0xFE4F | 2 |

每列计算最大显示宽度后，额外加 2 个空格间隔，用半角空格补齐。

### 3.5 编码检测（FileIO._looksGarbled）

读取文件时先以 UTF-8 解码，统计替换字符 `U+FFFD` 的比例。若超过 1%，则尝试用 GBK 重新解码，取乱码比例较低的版本。此机制用于兼容国内常见的 GBK 编码 CSV 文件。

## 4. 数据流

### 4.1 功能一：表格 → 纯文本

```
用户选择文件 (拖拽/点击)
  │
  ├─ .csv  → FileIO.readAsText() → Parsers.autoParse() → { headers, rows }
  └─ .xlsx/.xls → FileIO.readAsArrayBuffer() → ExcelIO.readExcel() → { headers, rows }
  │
用户选择输出格式 (CSV / Markdown / 固定宽度)
  │
  ├─ CSV:        Converters.toCSV(headers, rows)
  ├─ Markdown:   Converters.toMarkdown(headers, rows)
  └─ 固定宽度:   Converters.toFixedWidth(headers, rows)
  │
生成 Blob → Utils.triggerDownload()
```

### 4.2 功能二：纯文本 → Excel

```
用户选择文件 (拖拽/点击)
  │
FileIO.readAsText() → 原始文本
  │
FormatDetector.detect() → 格式信息
  │
Parsers.autoParse() → { headers, rows }
  │
TypeInferrer.inferColumnTypes() → [{ name, type, sample }]
  │
(UI 显示检测结果)
  │
ExcelIO.writeExcel(headers, rows, colTypes) → Blob
  │
Utils.triggerDownload()
```

## 5. UI 设计

### 5.1 布局

- 顶部：渐变色标题栏（蓝色）
- 主体：左右双栏弹性布局，每栏一个功能卡片
- 屏幕宽度 ≤860px 时自动切换为上下堆叠

### 5.2 设计规范

| 元素 | 样式 |
|------|------|
| 主色 | `#4A90D9` |
| 背景 | `#F0F6FC` |
| 卡片背景 | `#FFFFFF`，圆角 10px，阴影 |
| 拖拽区 | 2px 虚线边框 `#D6E4F0`，hover 时高亮为主色 |
| 按钮 | `#4A90D9` 背景白色文字，hover 加深，disabled 灰色 |
| 格式按钮 | 选中态：淡蓝背景 + 主色边框 |

- 拖拽悬停：拖拽区边框高亮 + 背景变色

## 6. SheetJS 调用说明

### 读取 Excel

```javascript
const wb = XLSX.read(arrayBuffer, { type: 'array' });
const sheet = wb.Sheets[wb.SheetNames[0]];
const data = XLSX.utils.sheet_to_json(sheet, { header: 1, defval: '' });
// data[0] = 表头行, data[1..n] = 数据行
```

### 写入 Excel

```javascript
const ws = XLSX.utils.aoa_to_sheet([headers, ...rows]);
// 逐列设置单元格类型
cell.t = 'n'; // 数值
cell.t = 'd'; cell.z = 'yyyy-mm-dd'; // 日期
cell.t = 's'; // 文本（默认）
const wb = XLSX.utils.book_new();
XLSX.utils.book_append_sheet(wb, ws, 'Sheet1');
const buf = XLSX.write(wb, { type: 'array', bookType: 'xlsx' });
```

## 7. 性能考量

| 场景 | 策略 |
|------|------|
| 大文件格式检测 | 仅分析前 20–50 行 |
| 大文件类型推断 | 最多采样前 100 行 |
| CSV 解析 | 逐行处理，一次性构建结果数组 |
| 固定宽度中文字符宽度 | 逐字符判断 Unicode 范围，时间复杂度 O(n) |

对于百 MB 级别的大文件，浏览器内存可能成为瓶颈。当前版本未实现流式处理，建议处理的文件大小不超过 50MB。

## 8. 安全性

- 所有文件处理在浏览器本地完成，无网络上传
- 无 `eval()` 或 `innerHTML` 的动态执行
- `innerHTML` 仅用于显示列类型标签（值来自受控的类型枚举，非用户输入）
- 下载文件通过 `URL.createObjectURL` + `<a>` 标签触发，不经过服务器
